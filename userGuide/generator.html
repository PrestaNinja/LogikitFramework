What Is the Application Generator?

Application generator is a script that generates all necessary files (the controller, the model, the form, the validator and the view files) with a single command.
The generated application will already have ajax capabilities for validation, sorting and pagination issues.
It will also recognize the field types and preset the relevant form components.
As as result: You will have a full-fledged CRUD application ready to use.



Example Application With the Application Generator

Application Generator assumes you already have AJAX enabled in your template file.
Examine Using AJAX section to learn more.



Preparing the Database Tables

Run the SQL statements below to create two sample tables: 
CREATE TABLE IF NOT EXISTS `entries` (
  `id` int(11) unsigned NOT NULL auto_increment,
  `title` char(100) NOT NULL,
  `entry` longtext NOT NULL,
  `date` date NOT NULL,
  PRIMARY KEY  (`id`)
) ENGINE=MyISAM  DEFAULT CHARSET=latin1 AUTO_INCREMENT=1 ;
	
CREATE TABLE IF NOT EXISTS `comments` (
  `id` int(11) unsigned NOT NULL auto_increment,
  `name` char(100) NOT NULL,
  `comment` longtext NOT NULL,
  PRIMARY KEY  (`id`)
) ENGINE=MyISAM DEFAULT CHARSET=latin1 AUTO_INCREMENT=1 ;

Setting the Environment

We will start with a new LF installation. Once you'll have an idea on how it works, you may invoke the generator anytime you need it.
  • install LF.
    Do not forget to edit application/system/config.php and application/system/database.ini to fit your database settings.
  • Open a command line and switch to your LF installation folder.
    Assuming your LF installation folder is /var/www/framework/:

    dev@logikit:~$ cd /var/www/framework
  • Make sure to have write permissions recursively for application folder:

    dev@logikit:/var/www/framework$ chmod -R 777 application



Generating the Code

  • Switch to system folder:

    dev@logikit:/var/www/framework$ cd system
  • Run the generator:

    dev@logikit:/var/www/framework/system$ ./generate.php
If your webroot is /var/www on your localhost, you can see your application running
http://localhost/framework/
