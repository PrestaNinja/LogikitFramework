Logikit::Framework URLs

URLs in LF are designed to be search-engine and human friendly. Rather than using the standard "query string" approach to URLs that is synonymous with dynamic systems, LF uses a segment-based approach:

Example URL: http://www.example.com/blog/display/id/12/showComments/1

  • The first segment represents the controller class that should be invoked.
  • The second segment represents the class function, or method, that should be called.
  • The additional segments, represent the variable names and values that will be passed to the controller.
    in the example above, the controller will take the value 12 for id and the value 1 for showComments variables.

LF will automatically tell if you are using a URL structure containing index.php or not.

http://www.example.com/index.php/blog/display/id/12/showComments/1
will be rendered exactly the same way as
http://www.example.com/blog/display/id/12/showComments/1
