What is a Model?

Models are PHP classes that are designed to work with information in your database.



Anatomy of a Model

Model classes are stored in your application/model/ folder. They can be nested within sub-folders if you want this type of organization.

The basic prototype for a model class is this: 

class ModelName extends LogikitModel
{
	public function __contruct()
	{
		parent::__contruct();
	}
    
}
Where ModelName is the name of your class. Class names must have the first letter capitalized. Make sure your class extends the base Model class.

The file name will be the same as your class name. For example, if your class is this: 

class UserModel extends LogikitModel
{
	public function __contruct()
	{
		parent::__contruct();
	}
    
}
Your file will be this:
application/model/UserModel.php



Loading a Model

Your models will typically be loaded and called from within your controller functions. To load a model you will use the following function: 

$this->load->model('ModelName');
If you model is located in a sub-folder, include the relative path from your models folder. For example, if you have a model located at application/model/Blog/QueryModel.php you will load it using: 
$this->load->model('Blog/QueryModel');
Once loaded, you will access your model functions using an object with the same name as your class: 
$this->load->model('ModelName');

$this->ModelName->function();
Here is an example of a controller, that loads a model, then serves a view: 
class Blog extends LogikitController
{

    public function index()
    {
		$data = array();
    
        $this->load->model('BlogModel');

        $data['query'] = $this->BlogModel->getLastTenEntries();

        $this->load->view('index', $data);
    }
}



Auto-loading Models

If you find that you need a particular model globally throughout your application, you can tell LF to auto-load it during system initialization. This is done by opening the application/config/autoload.php file and adding the model to the autoload array.
