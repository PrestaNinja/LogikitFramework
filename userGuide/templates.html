Templates

Templates are regular web pages that includes the static parts of your application: The header, the menu and the footer.
As they will include HTML headers, they also serve as the place where Javascripts and CSS files are loaded.



Creating a Template

Using your text editor, create an empty file and put this in it: 

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-frameset.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
<meta http-equiv="content-type" content="text/html; charset=utf-8" />
<title>Test page</title>
<meta name="keywords" content="" />
<meta name="description" content="" />
<link href="<?php echo URLROOT; ?>css/default.css" rel="stylesheet" type="text/css" />
</head>
<body>
	Header comes here.
	<p><?php $this->renderView(); ?></p>
	Footer comes here.
</body>
</html>
then rename the file application/template/main.php to something else (this is the default LF template file) not to overwrite it and save your new file as main.php in application/template folder.
We will use it right away together with a "view" described in the view section where we will put everything together and complete our first LF application.
as you are done with your template, you may proceed to the next step. 
<?php $this->renderView(); ?>
tells LF to render the view file and
URLROOT is the autogenerated constant telling the URL path of your LF installation.

Templates are required in a LF application and are automatically loaded.



Defining a Default Template

To specify a default template, open your application/config/config.php file and set this variable: 

$defaultTemplate = 'main';
Templates are found in /application/template folder.
main is the name of the template file you want used with a suffix ".php".
So, with the setting above, LF will look for application/template/main.php to load the template.
