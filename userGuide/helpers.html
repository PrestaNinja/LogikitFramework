Helper Functions

Helpers, as the name suggests, help you with tasks. Each helper file is simply a collection of functions in a particular category.

Helpers are not written in an Object Oriented format. They are simple, procedural functions. Each helper function performs one specific task, with no dependence on other functions.

LF does not load Helper Files by default (though there are a few exceptions) so the first step in using a Helper is to load it. Once loaded, it becomes globally available in your controller and views.



Loading a Helper

Loading a helper file is quite simple using the following function: 

$this->load->systemHelper('name');
Where name is the file name of the helper, without the .php file extension.
For example, to load the Ajax Helper file, which is named ajax.php, you would do this: 
$this->load->systemHelper('ajax');
A helper can be loaded anywhere within your controller functions (or even within your View files, although that's not a good practice), as long as you load it before you use it. You can load your helpers in your controller constructor so that they become available automatically in any function, or you can load a helper in a specific function that needs it.



Auto-loading Helpers

If you find that you need a particular helper globally throughout your application, you can tell LF to auto-load it during system initialization. This is done by opening the application/config/autoload.php file and adding the helper to the autoload array.



Using a Helper

Once you've loaded the Helper File containing the function you intend to use, you'll call it the way you would a standard PHP function. For example, to create a link using the siteLink() function in one of your view files you would do this: 

<?php echo siteLink('blog/comments', 'Click Here');?>
